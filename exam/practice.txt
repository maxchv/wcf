Написать сервис для тестирования студентов.
Сервис должен быть использован двумя типами клиентов.

Администратор имеет вмозможность:
	1. Указать название дисциплины
	2. Добавить вопросы с открытой формой ответов. 
           Все вопросы представлены в текстовом файле - по одному на строку.
	   Все вопросы должны сохранятся в базе данных на стороне сервиса.
	3. Указывает количество выдаваемых вопросов студентам.
	4. Просматривать результаты тестирования студентов.

Клиентское приложение для тестирования предусматривает следующий функционал:
	1. Студент указывать свое имя и фамилию, группу
	2. Получает заданное количество вопросов.
	3. Отвечает на вопросы (вводит данные в текстовое поле ввода)
	4. Сохраняет ответы в базе данных на сервисе.

Примечание:
	Всем студенты подключенные к сервису получают уникальные вопросы, т.е.
	у двух студентов не должны повторяться вопросы.

Дополнительное задание:
	Ограничить время проведения теста, для чего в административном приложении
	добавить возможность задавать время, а во время теста выводить время прохождения теста в
	обратном порядке.
